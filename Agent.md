# Agent 约定

本项目用于进行 GitHub 个人博客主页的项目开发，所有需求分析、页面设计、功能实现与相关调整，都应围绕该目标展开，避免偏离项目定位。

每一轮改动完成后，需要自动进行一次精简记录，更新当轮任务内容、已完成事项、当前进度、遗留问题与下一步计划，形成类似 memory 的轻量进度机制，便于后续连续协作和状态追踪。

开发过程中应做好 Git 管理，保持提交粒度清晰、提交信息明确，避免无关改动混入同一次提交；不得擅自回退、覆盖或清理他人的修改，发现工作区存在非本人变更时，应先识别影响范围，再在不破坏现有修改的前提下继续处理。

执行改动前必须先明确需求；如果需求描述、预期效果、影响范围或验收标准存在不清楚的地方，应先向用户提问并确认，再开始实施修改，避免基于猜测直接改动。
